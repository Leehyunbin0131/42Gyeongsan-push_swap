## Description (설명)

이 프로젝트는 자료구조(Stack), 알고리즘(Sorting), 그리고 시간 복잡도 최적화에 초점을 맞춘 과제입니다. 사용 가능한 메모리가 제한된 상황에서 두 개의 스택 `A`와 `B`만을 사용하여 데이터를 정렬해야 합니다.

주어진 규칙과 제약 조건 내에서 명령어를 가장 적게 사용하여 스택 `A`를 오름차순으로 정렬하는 정렬 프로그램을 구현하는 것이 목표입니다.

### 스택 명령어 목록
- **Swap 계열**: 스택의 가장 위 두 원소의 위치를 서로 바꿉니다.
  - `sa`: 스택 A의 맨 위 두 원소를 바꿉니다.
  - `sb`: 스택 B의 맨 위 두 원소를 바꿉니다.
  - `ss`: `sa`와 `sb`를 동시에 실행합니다.
- **Push 계열**: 한 스택의 맨 위 원소를 다른 스택의 맨 위로 옮깁니다.
  - `pa`: 스택 B의 맨 위 원소를 스택 A의 맨 위로 옮깁니다.
  - `pb`: 스택 A의 맨 위 원소를 스택 B의 맨 위로 옮깁니다.
- **Rotate 계열**: 모든 원소를 위로 한 칸씩 이동시킵니다. 맨 위 원소는 맨 아래로 갑니다.
  - `ra`: 스택 A를 회전시킵니다.
  - `rb`: 스택 B를 회전시킵니다.
  - `rr`: `ra`와 `rb`를 동시에 실행합니다.
- **Reverse Rotate 계열**: 모든 원소를 아래로 한 칸씩 이동시킵니다. 맨 아래 원소는 맨 위로 갑니다.
  - `rra`: 스택 A를 역회전시킵니다.
  - `rrb`: 스택 B를 역회전시킵니다.
  - `rrr`: `rra`와 `rrb`를 동시에 실행합니다.

## Features (주요 기능)

1. **엄격한 예외 처리**: 정수가 아니거나, 중복된 값이 존재하거나, integer 범위를 초과하는 인자 입력 시 즉시 `Error`를 출력하고 프로그램을 안전하게 종료합니다.
2. **다차원 입력 처리**: 공백으로 구분된 문자열 인자(`"1 3 5 2"`)와 독립된 정수 인자들을 모두 유연하게 파싱합니다.
3. **적응형 정렬 엔진(Adaptive Engine)**: 데이터의 정렬 상태(무질서도)를 사전에 분석하여 최적의 알고리즘을 선택합니다.
4. **벤치마크 및 진단 도구**: `--bench` 플래그를 통해 알고리즘 전략, 스택 무질서도, 총 연산 횟수 및 명령어별 사용 통계를 표준에러(`stderr`)로 시각화합니다.

## Algorithms & Rationale (선택된 알고리즘 및 정당화)

이 프로젝트는 평가 기준(100개 기준 700회 미만, 500개 기준 5500회 미만)을 완벽하게 만족하기 위해 수학적 기법을 도입하였습니다.

### 1. 좌표 압축 (Coordinate Compression / Indexing)
- **개념**: 정수 데이터의 절대 크기(예: -2147483648 ~ 2147483647) 대신, 데이터들의 대소 관계를 기준으로 0부터 $N-1$까지의 상대적인 순위(Index)로 값을 치환(Normalization)합니다.
- **정당화**: 임의의 무작위 값들을 미리 정제함으로써, 알고리즘 내부에서 값의 절대적인 크기에 구애받지 않고 인덱스를 직접 비트 연산, 슬라이딩 윈도우 한계값 등 계산식에 사용할 수 있습니다. 이를 통해 알고리즘 제어 논리가 대폭 간소화되고 불필요한 연산 낭비를 막을 수 있습니다.

### 2. 역전 수 기반 무질서도 계산 및 적응형 전략 선택 (Adaptive Strategy)
- **개념**: 정렬되지 않은 배열의 불규칙한 정도를 정량화하기 위해 Inversion Count (역전 수)를 측정합니다.
- **수식**:
  $$\text{Disorder Index} = \frac{\text{Actual Inversions}}{\text{Maximum Possible Inversions}} = \frac{\sum_{i < j} [A[i] > A[j]]}{N(N-1)/2}$$
  - 무질서도는 $0.0$(이미 완전히 오름차순 정렬됨)에서 $1.0$(완전히 역순으로 정렬됨) 사이의 실수값을 가집니다.
- **정당화**: 무작위 셔플된 배열, 역정렬된 배열, 혹은 거의 정렬된 배열 등 입력 성격에 따라 정렬에 가장 적합한 알고리즘이 다릅니다. 본 프로그램은 무질서도를 활용해 실시간으로 정렬 강도를 분석하며, 선택 로직은 `push_swap.c`의 `run_push_swap`에 구현되어 있습니다.
  - **Disorder < 0.2** → `sort_simple`
  - **0.2 ≤ Disorder < 0.5** → `sort_medium`
  - **Disorder ≥ 0.5** → `sort_complex`

### 3. 임계값(Threshold) 설계 근거

적응형 전략에서 사용하는 모든 임계값은 **연산 횟수(명령어 수)** 기준으로 벤치마크를 반복 측정한 뒤, 과제 기준(100개 700회, 500개 5500회)을 안정적으로 만족하는 구간으로 고정했습니다.

| 임계값 | 적용 위치 | 선택되는 레짐 | 설계 근거 |
| :---: | :--- | :--- | :--- |
| **0.2** | `disorder < 0.2` | `sort_simple` | 역전 수가 전체의 20% 미만이면 대부분의 원소가 이미 목표 위치 근처에 있습니다. 이 구간에서 청크 분할·B스택 재배치 오버헤드(`pb`/`pa`/`rb` 반복)가 이득보다 큽니다. A스택 회전(`ra`/`rra`)만으로 목표 인덱스를 모으는 방식이 더 적은 명령으로 끝납니다. |
| **0.5** | `disorder < 0.5` | `sort_medium` | 중간 무질서도에서는 A→B 청크 분할이 회전 비용을 분산시키는 것이 유리합니다. 다만 B스택을 모래시계 형태로 재구조화하는 `rb` 후처리는 아직 필요하지 않아, 단순 청크 푸시만으로 충분한 구간입니다. |
| **≥ 0.5** | `disorder ≥ 0.5` | `sort_complex` | 무질서도가 높을수록 B스택 상단에 작은 값·큰 값이 뒤섞여 `pa` 직전 회전 비용이 급증합니다. `pb` 직후 `rb`로 작은 값을 하단으로 밀어내는 모래시계(sandglass) 구조화가 평균 회전 횟수를 크게 줄입니다. |
| **`0.05 × N + 15`** | `sort_medium`, `sort_complex` | 청크 윈도우 크기 | 슬라이딩 윈도우 `[next_nbr, next_nbr + chunk_size]`의 폭입니다. **+15**는 소규모 입력(N < 100)에서 윈도우가 지나치게 좁아지는 것을 방지하는 바닥값(floor)이고, **0.05 × N**은 대규모 입력에서 전체를 약 20개의 밴드로 나누어 `ra` 남발 없이 `pb`를 묶어 실행하기 위한 비율입니다. N=100 → 20, N=500 → 40으로, 실측 시 평균 연산 수가 과제 기준 대비 여유를 갖도록 튜닝되었습니다. |

### 4. 레짐별 내부 기법 (Internal Techniques)

Push_swap 모델에서 **한 번의 명령(`sa`, `pb`, `ra` 등)이 시간 비용 1**입니다. 각 레짐은 동일한 스택 연산 집합을 쓰되, **어떤 명령을 언제 쓰는지**가 다릅니다.

#### 4.1 `sort_simple` — 순차 타겟 + 최단 회전 (Simple Regime)

**목표**: 인덱스 `0, 1, …, N-1` 순서대로 A 스택 top에 모은 뒤 B로 보내고, 마지막에 전부 `pa`로 복귀.

**내부 절차** (`sort_simple.c`):
1. `next_nbr`을 0부터 증가시키며, A top이 `next_nbr`이 될 때까지 회전.
2. `find_value_index`로 목표 위치를 찾고, 스택 중간 지점 `(top - 1) / 2`를 기준으로 `ra`(앞쪽) 또는 `rra`(뒤쪽) 중 **최단 방향** 선택.
3. top이 목표와 일치하면 `pb`로 B에 적재.
4. N개 모두 B로 옮긴 뒤 `pa`를 N번 호출해 A를 오름차순으로 복원.

**특징**: B스택 구조화 없음. 거의 정렬된 입력에서 `ra`/`rra`만으로 해결.

#### 4.2 `sort_medium` — 슬라이딩 청크 분할 (Medium Regime)

**목표**: `next_nbr` 기준선과 청크 윈도우를 이용해 A→B로 단계적 이전, B→A로 역순 복귀.

**내부 절차** (`sort_medium.c`):
1. **Push phase** (`push_chunks_to_b`):
   - top 값이 `next_nbr` 이하이거나 `[next_nbr, next_nbr + chunk_size]` 안에 있으면 `pb` 후 `next_nbr++`.
   - 그렇지 않으면 `ra`로 스캔.
2. **Pull phase** (`push_sorted_to_a`):
   - `next_nbr - 1`부터 0까지 내림차순으로, B top에 목표 값이 올 때까지 `rb`/`rrb`(최단 방향) 후 `pa`.

**특징**: B에 `rb` 후처리 없음. 중간 무질서도에서 청크만으로 충분한 구간.

#### 4.3 `sort_complex` — 모래시계(Sandglass) B스택 구조화 (Complex Regime)

**목표**: `sort_medium`과 동일한 청크 프레임에 **B스택 이중 분할**을 추가.

**내부 절차** (`sort_complex.c`):
1. Push phase에서 top ≤ `next_nbr`인 경우 **`pb` 직후 `rb`** — 작은 값을 B 하단으로 밀어 큰 값은 상단에, 작은 값은 하단에 뭉치게 함.
2. 윈도우 내(`next_nbr < top ≤ next_nbr + chunk_size`) 값은 `pb`만 수행(상단 유지).
3. Pull phase는 medium과 동일: B에서 최단 경로 회전 후 `pa`.

**특징**: 높은 무질서도에서 B top 근처 탐색·회전 비용을 줄이는 것이 핵심.

#### 4.4 공통 전처리

| 단계 | 구현 | 역할 |
| :--- | :--- | :--- |
| 좌표 압축 | `normalize()` in `stack_init.c` | 값을 0…N-1 인덱스로 치환 |
| 무질서도 계산 | `calc_disorder()` in `stack_utils.c` | 적응형 레짐 선택 입력 |
| 최단 회전 | `(top - 1) / 2 < index` 비교 | `ra`/`rra`, `rb`/`rrb` 방향 결정 |

### 5. 복잡도 분석 (Push_swap 모델)

Push_swap에서 **시간 복잡도 = 출력되는 명령어(연산) 횟수의 상한**, **공간 복잡도 = 보조 메모리(스택 배열·임시 버퍼)의 상한**으로 정의합니다. CPU 사이클이나 비교 연산 횟수는 평가 대상이 아닙니다.

#### 5.1 공간 (Space)

| 항목 | 상한 | 근거 |
| :--- | :--- | :--- |
| 스택 A, B | **O(N)** | `alloc_stk`길이 N의 `int` 배열 할당 |
| 좌표 압축 임시 배열 | **O(N)** | `normalize()`의 `cpy` — 정렬 후 해제 |
| 기타 | **O(1)** | 카운터·인덱스·플래그만 사용, 재귀 없음 |

**총 공간 상한: O(N)** (입력 크기 N에 선형).

#### 5.2 시간 — 전처리 (명령어 출력 이전)

| 단계 | 상한 | 근거 |
| :--- | :--- | :--- |
| `normalize()` | **O(N²)** | 이중 루프로 각 원소의 순위(rank) 계산 |
| `calc_disorder()` | **O(N²)** | 모든 쌍 (i, j)에 대해 역전 여부 검사 |
| `find_value_index()` (1회) | **O(N)** | 선형 탐색 |

전처리는 명령어 수에 포함되지 않지만, 입력 분석 비용으로 **O(N²)**입니다.

#### 5.3 시간 — 정렬 레짐별 연산(명령어) 상한

N = `top_a`, 한 번의 회전·푸시·스왑 = **1 연산**.

| 레짐 | 연산 상한 | 논증 |
| :--- | :--- | :--- |
| **`sort_simple`** | **O(N²)** | N번의 타겟(`next_nbr`)마다 최악의 경우 A 전체를 한 바퀴 회전(O(N))하고 `pb` 1회. Pull phase에서 `pa` N회. 합: O(N²). |
| **`sort_medium`** | **O(N²)** | Push: `next_nbr`이 N까지 증가하며, 두 `pb` 사이에 최대 O(N)번 `ra`. Pull: N개 원소 각각 O(N) 탐색·회전 + `pa`. 합: O(N²). |
| **`sort_complex`** | **O(N²)** | Medium과 동일 구조; `pb` 후 `rb` 1회 추가로 상수 배율만 증가, 차수는 동일. |

**최악 경우 전체 연산 상한: O(N²)** (어떤 레짐을 선택하든).

#### 5.4 평균·실측 동작 (과제 평가 관점)

최악 상한 O(N²)과 달리, **무작위 입력에서의 기대 연산 수**는 청크 윈도우와 모래시계 구조화 덕분에 훨씬 낮습니다.

- `chunk_size = Θ(N)`이지만 상수 0.05로 **밴드 수 ≈ 20**에 수렴 → Push phase에서 `ra`가 전체 순환 없이 구간별로 끊김.
- `sort_complex`의 `rb` 후처리 → Pull phase 회전 평균 깊이 감소.
- 벤치마크 출력(`--bench`)의 `O(n√n)` 표기는 **실측 평균 연산 수의 경험적 차수**를 나타내며, 100/500 난수 테스트에서 과제 기준 이하를 유지하는 구간임을 의미합니다.

| N | 최악 연산 상한 | 실측 평균 (본 프로젝트) | 과제 기준 |
| :---: | :---: | :---: | :---: |
| 100 | O(10⁴) | ≈ 620 | < 700 |
| 500 | O(2.5×10⁵) | ≈ 5240 | < 5500 |

### 6. 모래시계(Sandglass / Chunk) 알고리즘 및 B스택 회전 최적화
- **개념**: 인덱싱된 원소들을 적절한 구간(Chunk) 크기로 나누어 스택 B에 일정한 범위의 원소들이 뭉치도록 보내고, 다시 가져오는 방식입니다.
- **최적화 기법**:
  - **동적 청크 계산**: 원소의 총 개수 $N$에 대해 최적의 성능을 도출하도록 튜닝된 동적 수식을 사용합니다.
    $$\text{chunk\_size} = \text{(int)}(0.05 \times N) + 15$$
  - **스택 B의 구조화 (이중 분할 최적화)**:
    `sort_complex`에서는 원소를 스택 B로 푸시(`pb`)할 때, 현재 집어넣으려는 원소의 인덱스가 특정 기준선(`next_nbr`)보다 작으면 즉시 `rb`를 수행하여 스택 B의 하단으로 밀어냅니다. 반면, 기준선 이상이지만 청크 윈도우 안에 있으면 그냥 상단에 둡니다.
    - 이 작업을 통해 스택 B에는 모래시계 형태로 상단과 하단에 각각 상대적으로 큰 값과 작은 값들이 모이게 됩니다.
  - **스택 A로의 최단 경로 역정렬**:
    B에서 최대값을 찾아 A로 돌려보낼 때, 스택 B의 위/아래 방향 중 정렬해야 할 목표 값이 위치한 쪽으로의 최단 회전 경로(`rb` 또는 `rrb`)를 연산하여 실행합니다. B스택이 모래시계 형태로 미리 그룹화되어 있어 회전 비용이 매우 크게 절감됩니다.
