## Description (설명)

이 프로젝트는 자료구조(Stack), 알고리즘(Sorting), 그리고 시간 복잡도 최적화에 초점을 맞춘 과제입니다. 사용 가능한 메모리가 제한된 상황에서 두 개의 스택 `A`와 `B`만을 사용하여 데이터를 정렬해야 합니다.

주어진 규칙과 제약 조건 내에서 명령어를 가장 적게 사용하여 스택 `A`를 오름차순으로 정렬하는 정렬 프로그램을 구현하는 것이 목표입니다.

### 스택 명령어 목록
- **Swap 계열**: 스택의 가장 위 두 원소의 위치를 서로 바꿉니다.
  - `sa`: 스택 A의 맨 위 두 원소를 바꿉니다.
  - `sb`: 스택 B의 맨 위 두 원소를 바꿉니다.
  - `ss`: `sa`와 `sb`를 동시에 실행합니다.
- **Push 계열**: 한 스택의 맨 위 원소를 다른 스택의 맨 위로 옮깁니다.
  - `pa`: 스택 B의 맨 위 원소를 스택 A의 맨 위로 옮깁니다.
  - `pb`: 스택 A의 맨 위 원소를 스택 B의 맨 위로 옮깁니다.
- **Rotate 계열**: 모든 원소를 위로 한 칸씩 이동시킵니다. 맨 위 원소는 맨 아래로 갑니다.
  - `ra`: 스택 A를 회전시킵니다.
  - `rb`: 스택 B를 회전시킵니다.
  - `rr`: `ra`와 `rb`를 동시에 실행합니다.
- **Reverse Rotate 계열**: 모든 원소를 아래로 한 칸씩 이동시킵니다. 맨 아래 원소는 맨 위로 갑니다.
  - `rra`: 스택 A를 역회전시킵니다.
  - `rrb`: 스택 B를 역회전시킵니다.
  - `rrr`: `rra`와 `rrb`를 동시에 실행합니다.

## Features (주요 기능)

1. **엄격한 예외 처리**: 정수가 아니거나, 중복된 값이 존재하거나, integer 범위를 초과하는 인자 입력 시 즉시 `Error`를 출력하고 프로그램을 안전하게 종료합니다.
2. **다차원 입력 처리**: 공백으로 구분된 문자열 인자(`"1 3 5 2"`)와 독립된 정수 인자들을 모두 유연하게 파싱합니다.
3. **적응형 정렬 엔진(Adaptive Engine)**: 데이터의 정렬 상태(무질서도)를 사전에 분석하여 최적의 알고리즘을 선택합니다.
4. **벤치마크 및 진단 도구**: `--bench` 플래그를 통해 알고리즘 전략, 스택 무질서도, 총 연산 횟수 및 명령어별 사용 통계를 표준에러(`stderr`)로 시각화합니다.


### 커맨드 라인 플래그 (옵션)

본 프로그램은 성능 분석 및 강제 정렬 전략 적용을 위한 고유의 플래그를 제공합니다:

- `--simple`: 강제로 단순 정렬(O(N²))을 수행합니다. (원소가 적거나 이미 정렬도가 높은 경우 적합)

## Algorithms & Rationale (선택된 알고리즘 및 정당화)

이 프로젝트는 평가 기준(100개 기준 700회 미만, 500개 기준 5500회 미만)을 완벽하게 만족하기 위해 수학적 기법을 도입하였습니다.

### 1. 좌표 압축 (Coordinate Compression / Indexing)
- **개념**: 정수 데이터의 절대 크기(예: -2147483648 ~ 2147483647) 대신, 데이터들의 대소 관계를 기준으로 0부터 $N-1$까지의 상대적인 순위(Index)로 값을 치환(Normalization)합니다.
- **정당화**: 임의의 무작위 값들을 미리 정제함으로써, 알고리즘 내부에서 값의 절대적인 크기에 구애받지 않고 인덱스를 직접 비트 연산, 슬라이딩 윈도우 한계값 등 계산식에 사용할 수 있습니다. 이를 통해 알고리즘 제어 논리가 대폭 간소화되고 불필요한 연산 낭비를 막을 수 있습니다.

### 2. 역전 수 기반 무질서도 계산 및 적응형 전략 선택 (Adaptive Strategy)
- **개념**: 정렬되지 않은 배열의 불규칙한 정도를 정량화하기 위해 Inversion Count (역전 수)를 측정합니다.
- **수식**:
  $$\text{Disorder Index} = \frac{\text{Actual Inversions}}{\text{Maximum Possible Inversions}} = \frac{\sum_{i < j} [A[i] > A[j]]}{N(N-1)/2}$$
  - 무질서도는 $0.0$(이미 완전히 오름차순 정렬됨)에서 $1.0$(완전히 역순으로 정렬됨) 사이의 실수값을 가집니다.
- **정당화**: 무작위 셔플된 배열, 역정렬된 배열, 혹은 거의 정렬된 배열 등 입력 성격에 따라 정렬에 가장 적합한 알고리즘이 다릅니다. 본 프로그램은 무질서도를 활용해 실시간으로 정렬 강도를 분석합니다:
  - **Disorder < 0.2**: 단순 인플레이스(In-place) 삽입/회전 위주의 `sort_simple`을 선택하여 불필요하게 스택 B로 많은 원소를 보내는 명령 손실을 줄입니다.
  - **Disorder < 0.5**: `sort_medium`을 선택하여 효율적인 청크 정렬을 수행합니다.
  - **Disorder >= 0.5**: 극도로 정렬되지 않은 상태이므로 B스택 분할 최적화가 적용된 `sort_complex`를 선택합니다.

### 3. 모래시계(Sandglass / Chunk) 알고리즘 및 B스택 회전 최적화
- **개념**: 인덱싱된 원소들을 적절한 구간(Chunk) 크기로 나누어 스택 B에 일정한 범위의 원소들이 뭉치도록 보내고, 다시 가져오는 방식입니다.
- **최적화 기법**:
  - **동적 청크 계산**: 원소의 총 개수 $N$에 대해 최적의 성능을 도출하도록 튜닝된 동적 수식을 사용합니다.
    $$\text{chunk\_size} = \text{(int)}(0.05 \times N) + 15$$
  - **스택 B의 구조화 (이중 분할 최적화)**:
    `sort_complex`에서는 원소를 스택 B로 푸시(`pb`)할 때, 현재 집어넣으려는 원소의 인덱스가 특정 기준선(`next_nbr`)보다 작으면 즉시 `rb`를 수행하여 스택 B의 하단으로 밀어냅니다. 반면, 기준선 이상이지만 청크 윈도우 안에 있으면 그냥 상단에 둡니다.
    - 이 작업을 통해 스택 B에는 모래시계 형태로 상단과 하단에 각각 상대적으로 큰 값과 작은 값들이 모이게 됩니다.
  - **스택 A로의 최단 경로 역정렬**:
    B에서 최대값을 찾아 A로 돌려보낼 때, 스택 B의 위/아래 방향 중 정렬해야 할 목표 값이 위치한 쪽으로의 최단 회전 경로(`rb` 또는 `rrb`)를 연산하여 실행합니다. B스택이 모래시계 형태로 미리 그룹화되어 있어 회전 비용이 매우 크게 절감됩니다.

### AI 활용에 대한 설명
이 프로그램은 교육과정의 목표에 맞게 스스로 공부하고 문제를 해결하는 것을 원칙으로 삼았습니다. 인공지능은 코드를 대신 짜주는 도구가 아니라, 원리를 가르쳐주는 보조 선생님이자 조언자로만 올바르게 활용했습니다.

1. **정렬 원리 공부와 공식 만들기**:
   - 모래시계 정렬 방식이 왜 많은 숫자를 정렬할 때도 효과적인지 원리를 이해하는 데 도움을 받았습니다.
   - 무작위로 뒤섞인 숫자들이 얼마나 어지럽게 꼬여있는지 숫자로 계산해 내는 공식을 만들고, 이 공식이 여러 상황에서 올바르게 작동하는지 검증하는 과정에서 자문을 구했습니다.

3. **오류 수정과 특이한 입력 해결**:
   - 정렬하는 도중에 숫자의 순서가 꼬이거나 프로그램이 갑자기 멈춰버리는 정밀한 논리 오류들을 추적하고 고쳐나갔습니다.
   - 숫자가 한 줄로 길게 묶여서 들어오거나, 허용된 범위를 벗어난 비정상적인 값이 들어오는 등 특이한 상황에서도 프로그램이 꺼지지 않고 안전하게 에러를 띄우도록 예외 처리 흐름을 완성했습니다.
